### 5. 销售详情页面

**显示内容：**

1. **顶部信息**
   - 店铺名称和等级

2. **每日数据表格**
   - 表头：天数、进货额度、进货成本、每日收益、每日利润、补货后余额、累计回款、累计利润
   
   **计算逻辑：**
   
   **每日进货成本计算：**
   ```
   if (剩余预算 >= 每日进货额度) {
     今日进货成本 = 每日进货额度
   } else {
     今日进货成本 = 剩余预算（全部用于进货）
   }
   ```
   
   **每日收益计算：**
   ```
   每日收益 = 今日进货成本 * (1 + 利润率)
   ```
   
   **每日利润计算：**
   ```
   每日利润 = 每日收益 - 今日进货成本
   ```
   
   **补货后余额计算：**
   ```
   if (有余额利润) {
     补货后余额 = 余额利润 + 今日进货成本
   } else {
     补货后余额 = 今日进货成本
   }
   ```
   
   **累计回款计算：**
   ```
   if (天数 >= 12) {
     累计回款 = 累计回款 + 每日进货成本
   } else {
     累计回款 = 0
   }
   ```
   
   **累计利润计算：**
   ```
   if (天数 >= 12) {
     累计利润 = 累计利润 + 每日利润
   } else {
     累计利润 = 累计利润 + 每日利润（未回款部分）
   }
   ```

3. **底部汇总**
   - 总天数
   - 总进货成本
   - 总收益
   - 总利润
   - 累计回款

## 关键业务逻辑

### 1. 店铺数据结构

```typescript
interface ShopLevel {
  id: string;
  name: string;
  color: string; // 店铺颜色（青色、蓝灰、金黄、天蓝、青绿、黑色、紫色）
  startFund: number; // 开店资金
  dailyStock: number; // 每日进货额度
  dailySellRatio: number; // 每日卖出比例（1.0 = 100%）
  dailyProfit: number; // 每日利润
  bonusRatio: number; // 差额奖提成比例
}

const shopLevels: ShopLevel[] = [
  {
    id: 'bronze',
    name: '青铜店',
    color: 'cyan',
    startFund: 500,
    dailyStock: 500,
    dailySellRatio: 1.0,
    dailyProfit: 50,
    bonusRatio: 0.2
  },
  // ... 其他6个等级
];
```

### 2. 进货计算逻辑

**核心原则：**
- 每日卖出额度 = dailyStock * dailySellRatio
- 预算够多（无空档期）：只补货到刚好够第二天卖的额度
- 预算不足（有空档期）：用所有可用资金进货

**计算流程：**

```
输入：
- 用户选择的店铺等级
- 用户预算
- 每日推荐额度（可选）

初始化：
- remainingBudget = 用户预算
- totalProfit = 0
- totalReturn = 0
- dailyReturns = []

每天计算（第1天到第N天）：
1. 计算今日进货成本
   if (remainingBudget >= dailySellAmount) {
     todayStockCost = dailySellAmount
   } else {
     todayStockCost = remainingBudget（全部用于进货）
   }
   
2. 计算今日收益
   todayRevenue = todayStockCost * (1 + profitRate)
   
3. 计算今日利润
   todayProfit = todayRevenue - todayStockCost
   
4. 更新累计利润
   totalProfit += todayProfit
   
5. 更新剩余预算
   remainingBudget -= todayStockCost
   
6. 计算回款（第12天开始）
   if (day >= 12) {
     totalReturn += todayStockCost
   }
   
7. 添加到每日数据列表
   dailyReturns.push({
     day,
     todayStockCost,
     todayRevenue,
     todayProfit,
     remainingBudget,
     totalReturn,
     totalProfit
   })
```

### 3. 利润率计算

**每日利润率：**
```
利润率 = 每日利润 / 每日进货成本
```

**示例：**
- 青铜店：50 / 500 = 10%
- 白银店：110 / 1,000 = 11%
- 黄金店：625 / 5,000 = 12.5%

### 4. 回款机制

**回款规则：**
- 前11天：不回款，资金用于进货和复利增长
- 从第12天开始：每日进货成本全额回款

**回款公式：**
```
if (day >= 12) {
  今日回款 = 今日进货成本
} else {
  今日回款 = 0
}

累计回款 = sum(今日回款 for day = 1 to N)
```

### 5. 复利增长逻辑

**适用场景：**
- 预算不足时，用所有可用资金进货
- 利润用于继续进货，实现利滚利

**不适用场景：**
- 预算够多时，只补货到刚好够第二天卖的额度
- 达到循环后不再增加利润

### 2. Service Worker配置

**缓存策略：**
- 缓存所有静态资源（HTML、JS、CSS、图片）
- 使用Cache-first策略
- 版本号控制缓存更新

**关键代码：**
```javascript
const CACHE_NAME = 'app-cache-v1.5.4';

self.addEventListener('install', (event) => {
  event.waitUntil(
    caches.open(CACHE_NAME).then((cache) => {
      return cache.addAll([
        '/',
        '/manifest.json',
        '/icon-192.png',
        '/icon-512.png'
      ]);
    })
  );
});

self.addEventListener('fetch', (event) => {
  event.respondWith(
    caches.match(event.request).then((response) => {
      return response || fetch(event.request);
    })
  );
});
```

async function copyToClipboard(text: string): Promise<boolean> {
  try {
    // 现代方法
    await navigator.clipboard.writeText(text);
    return true;
  } catch (err) {
    // 降级方案
    const textarea = document.createElement('textarea');
    textarea.value = text;
    textarea.setAttribute('readonly', '');
    textarea.style.position = 'absolute';
    textarea.style.left = '-9999px';
    document.body.appendChild(textarea);
    textarea.select();
    try {
      document.execCommand('copy');
      document.body.removeChild(textarea);
      return true;
    } catch (err) {
      document.body.removeChild(textarea);
      return false;
    }
  }
}
